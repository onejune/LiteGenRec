# Exp10: 位置编码消融实验

## 实验背景

在 Exp03 (FullTransformer V2) 中使用了可学习的位置编码。但 CTR 特征列是否真的有"位置"语义？本实验通过消融研究验证。

## 数据集说明

### Criteo Display Advertising Challenge

- **来源**: Kaggle Competition (2014)
- **任务**: 预测用户是否会点击广告

### 本实验使用的数据

| 项目 | 值 |
|------|-----|
| **数据集名称** | Criteo Standard |
| **训练集样本数** | 1,345,295 |
| **验证集样本数** | 75,071 |
| **类别特征数** | 26 个 (C1-C26) |
| **数值特征数** | 13 个 (I1-I13)，本实验未使用 |
| **标签分布** | 点击率约 25% |

### 数据路径

```
/mnt/data/oss_wanjun/pai_work/open_research/dataset/criteo_standard/criteo-parquet/
```

### 数据预处理

与 exp03 保持一致：
- 类别特征使用 LabelEncoder 编码
- 缺失值填充为 `__MISSING__`
- 实验3 中随机打乱特征列顺序

## 实验设计

| 实验 | 设置 | 目的 |
|------|------|------|
| 实验1 | 有位置编码 (原始) | 基准 |
| 实验2 | 无位置编码 | 验证位置编码是否有益 |
| 实验3 | 随机打乱特征顺序 + 位置编码 | 验证位置编码学到了什么 |

| 参数 | 值 |

## 实验结果

| 实验 | AUC | 相对差异 |
|------|-----|----------|
| 有位置编码 (原始) | 0.7438 | 基准 |
| **无位置编码** | **0.7454** | **+1.6bp ✅** |
| 打乱特征顺序 | 0.7388 | -5.0bp |

## 关键发现

### 1. 位置编码有害 ❌

```
有位置编码: AUC 0.7438
无位置编码: AUC 0.7454
差异: +1.6bp (去掉位置编码更好)
```

**结论**: CTR 特征列没有顺序语义，位置编码引入了噪声

### 2. 打乱特征顺序有影响

```
原始顺序: AUC 0.7438
打乱顺序: AUC 0.7388
差异: -5.0bp
```

**分析**: 打乱顺序后性能下降，但这不是因为"位置语义"，而是因为:
- 模型学到了一些与特征列顺序相关的模式
- 这些模式不是真正的位置语义，而是训练数据中的偶然模式

### 3. 理论分析

**为什么位置编码在 NLP 有效但在 CTR 无效？**

| 场景 | 特征顺序 | 位置语义 |
|------|----------|----------|
| NLP | 文本序列 | ✅ 有 (词的位置有意义) |
| CTR | 特征列 | ❌ 无 (第1列和第5列没有"先后"概念) |

**CTR 特征的顺序是任意的**:
- C1=用户ID, C2=广告ID 还是 C1=广告ID, C2=用户ID
- 这个顺序对模型语义没有影响
- 强加位置编码反而引入噪声

## 建议

1. **移除位置编码**: V3 版本应去掉位置编码
2. **使用 Field Embedding**: 可学习的特征域嵌入替代位置编码
3. **保持特征顺序一致性**: 训练和推理时特征顺序要一致
